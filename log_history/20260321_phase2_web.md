## 任務清單

### 1. [v] 端到端驗收測試
- 檔案：`web_mahjong.py`、`mahjong.py`
- Commit: 本次驗收，無需修改（HTTP 各端點均正常，JSON 序列化無誤）
- 摘要：
  - 啟動 uvicorn，確認 `/`、`/new_game`、`/discard`、`/action` 全部回傳正確格式
  - 模擬一局完整流程（start → human_discard → prompt → game_over）
  - 找出並修復 JSON 序列化問題（PromptInfo 巢狀 dataclass 等）

### 2. [v] WebSocket AI 回合 log 串流
- 檔案：`web_mahjong.py`、`static/app.js`
- 摘要：
  - 在 `web_mahjong.py` 新增 `WebSocket /ws`
  - AI 回合每步透過 WS 推送 log，前端逐行附加至 log-box
  - 取代目前「整批回傳全部 log」的作法，增加即時感

### 3. [v] 麻將牌 Unicode 圖形化
- 檔案：`static/app.js`、`static/style.css`
- 摘要：
  - 利用 Unicode U+1F000–U+1F02B 麻將符號替換牌面文字
  - 維持 data-suit 花色著色，tooltip 顯示牌名
  - 目標：牌片視覺接近 example.webp 的貼圖風格

### 4. [v] 連莊 / 再來一局機制
- 檔案：`web_mahjong.py`、`static/app.js`
- 摘要：
  - 遊戲結束畫面新增「連莊」與「新局」按鈕
  - `/new_game` 接受 `dealer_idx` 與 `consecutive` 參數，傳給 `GameSession`
  - 前端顯示連莊次數、局風進圈（東→南→西→北）

### 4b. [v] 修正：「下莊」按鈕改為重啓遊戲
- 檔案：`static/app.js`
- 摘要：
  - 目前「下莊繼續」會帶入勝者為新莊家繼續對局，行為不符預期
  - 改為：下莊時只顯示「新局」按鈕，呼叫 `_startNewGame()`（完全重啓）
  - 「連莊」按鈕行為不變（莊家不動、consecutive+1）

### 5. [v] 手機響應式排版
- 檔案：`static/style.css`、`static/index.html`
- 摘要：
  - `@media (max-width: 600px)`：對家/上家/下家改為折疊顯示
  - 你的手牌縮小為 2 列，防止溢出
  - log-box 改為底部抽屜（可展開收合）
